# UI Field Importer (`ui_import_fields.py`)

This script provides a "surgical strike" method for updating specific translations without manually editing JSON files. It is ideal for importing translations generated by chatbots or spreadsheets.

## Core Features
*   **Anchor-Based Positioning**: Allows you to specify where a new key should be inserted (e.g., after a specific `___section` header).
*   **Batch Updates**: Processes a single CSV file to update multiple languages simultaneously.
*   **Smart Overwrite**: If a key already exists, the script updates the translation text while maintaining its current position in the file.
*   **Auto-Validation**: Automatically triggers the `ui_validate_locales.py` script after every import to ensure no errors were introduced.

## CSV Format
The script expects a file named `import_fields.csv` with the following structure:
| Key | Type/Lang | Content |
| :--- | :--- | :--- |
| `my_new_key` | `anchor` | `___section: settings.js` |
| `my_new_key` | `en` | `Settings Header` (Ignored - for reference) |
| `my_new_key` | `fr` | `Paramètres` |

## How to Use
1. Create your `import_fields.csv` with your new translations.
2. Run the script:
   
```bash
   python3 ui_import_fields.py
   ```
3. The script will report which keys were "Added" or "Updated" and then show the final validation status.
